# 数据库初始化脚本说明

## 脚本文件
- `init-database.sql` - 完整的数据库初始化脚本

## 功能说明
1. 创建数据库和所有表结构
2. 插入默认的预警配置数据
3. 插入测试数据用于开发调试
4. 创建数据库用户和权限
5. 输出初始化结果统计

## 表结构概览

### 核心业务表
1. **companies** - 相关方单位表
   - 存储单位基础信息
   - 包含单位类型、状态、风险等级等字段
   - 支持软删除和审计字段

2. **company_certificates** - 单位证照表
   - 存储单位的各类证照信息
   - 支持证照状态跟踪和审核流程
   - 包含文件上传相关字段

3. **certificate_alerts** - 证照预警配置表
   - 配置不同级别的预警规则
   - 支持多种通知方式（邮件、短信、微信等）

4. **certificate_alert_logs** - 证照预警记录表
   - 记录预警发送历史
   - 跟踪通知状态和接收人

5. **file_storage** - 文件存储表
   - 统一管理所有上传的文件
   - 支持多种存储方式（本地、S3、OSS、COS）
   - 包含文件去重和访问控制

## 使用说明

### 1. 初始化数据库
```bash
# 使用MySQL客户端执行
mysql -u root -p < scripts/init-database.sql
```

### 2. 验证初始化结果
```sql
-- 查看所有表
SHOW TABLES;

-- 查看表结构
DESC companies;

-- 查看测试数据
SELECT * FROM companies;
SELECT * FROM company_certificates;
```

### 3. 重置数据库
```bash
# 删除并重新创建
mysql -u root -p -e "DROP DATABASE IF EXISTS industrial_partner_management;"
mysql -u root -p < scripts/init-database.sql
```

## 注意事项

1. **生产环境**：请修改默认密码和用户权限
2. **数据安全**：敏感数据建议加密存储
3. **备份策略**：定期备份数据库和文件
4. **性能优化**：根据数据量调整索引策略
5. **权限控制**：严格限制数据库访问权限
